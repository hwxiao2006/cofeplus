# 故障管理页桌面端表格式重设计

## 背景
当前 `faults.html` 已完成“只保留远程操作按钮、支持设备搜索、去除状态 Tab”等交互收敛，但桌面端仍以卡片分段呈现，信息密度低，在异常记录较多时阅读效率不足。

## 目标
- 仅重设计桌面端视觉样式，不改动现有交互逻辑。
- 列表改为信息密集的运维控制台表格式布局。
- 保留现有核心行为：搜索设备、仅展示故障设备、仅保留“远程操作”按钮。
- 移动端保持当前可用体验，不做功能扩展。

## 已确认方案
用户已确认采用 A 风格 + 方案 1（表格式故障清单）：
- 顶部工具区：搜索框 + 统计信息。
- 主体为表格式清单：`设备编号 | 设备地址 | 故障内容 | 故障发生时间 | 操作`。
- 操作列仅保留 `远程操作`。
- 行 hover、斑马对比、边框分隔提升可读性。

## 结构设计
1. 保留页面现有外层布局（侧边栏、头部、内容区）。
2. 将故障列表主渲染结构调整为：
   - 表头容器（固定字段名）
   - 表体容器（逐行设备数据）
3. 在窄屏下将表格行转为堆叠展示，避免破坏移动端。

## 样式策略
- 整体采用深浅对比的运维控制台风格：浅灰背景 + 白色表格容器 + 清晰分隔线。
- 表头使用弱强调底色与较小字重，突出“数据区”而非装饰。
- 故障内容字段允许折行，时间字段保持单行。
- 操作按钮缩小为表格行内按钮风格，避免视觉抢占。

## 兼容与边界
- 无数据时仍显示“暂无故障记录”空状态。
- 保持远程操作弹层逻辑不变（桌面弹窗 / 移动底部弹层）。
- 原有状态记录、异常记录、运维记录的脚本函数继续保留，避免影响既有测试与后续入口。

## 验收标准
- 桌面端（>=1024）故障列表呈现为表格式，不再是大卡片字段块。
- 页面仍只显示故障设备，搜索可按设备编号/点位名过滤。
- 每行仅有一个“远程操作”按钮，且能正常打开操作弹层。
- 既有 `tests/faults.behavior.test.js` 通过。
