# 故障页当前设备运维记录设计

## 背景
当前故障页 `faults.html` 的“状态记录”弹层中，“运维记录”仍是“开发中”提示，无法查看实际记录。运维同学需要在当前设备上下文中快速查看该设备的操作历史。

## 目标
- 在“状态 -> 运维记录”中展示当前设备的运维记录列表。
- 保持交互模式与“异常记录”一致（桌面端对话层、移动端全屏层）。
- 仅展示当前设备记录，不做全设备汇总。

## 方案（已确认 B）
- 点击“运维记录”后，进入二级页面（复用 overlay 结构），标题为 `${deviceId}-运维记录`。
- 列表字段：时间、操作人、操作项、结果、备注。
- 返回逻辑：
  - 运维记录页返回 -> 状态页
  - 关闭状态页 -> 回到设备卡片

## 数据结构（前端原型）
使用 `getOperationRecords(deviceId)` 返回 mock 数据：

```js
{
  time: "2026-03-04 10:25:17",
  operator: "王天宇",
  action: "远程重启",
  result: "成功",
  note: "重启后制冰机恢复"
}
```

## 异常与边界
- 无记录时显示空状态“暂无运维记录”。
- 字段缺失显示 `-`。
- 文案过长时列表中折行展示。

## 验收标准
- “状态”页点击“运维记录”可打开记录页。
- 标题包含当前设备编号。
- 列表能看到“操作人/操作项/结果”信息。
- 返回按钮能回到“状态”页。
