# Device Entry 页面双端适配设计

> 日期：2026-02-28  
> 页面：`/Users/tigerhuang/cofeplus/device-entry.html`

## 1. 背景
当前 `device-entry` 页面基于移动端单列表单构建，桌面端仅放大容器宽度，导致信息稀疏、视线跨度大、操作区域分散，不利于桌面场景快速录入。

## 2. 目标
- 桌面端（>=1024px）升级为两栏布局，提高信息密度和录入效率。
- 移动端保留单栏流程，并做轻量体验优化（间距、输入对齐、上传区与按钮可用性）。
- 不改业务逻辑，不改字段 ID，不影响现有脚本行为。

## 3. 设计决策（已确认）
1. 桌面采用两栏布局（Section 级分栏，不做字段级重排）。
2. 移动端仅做轻量优化，不重做交互流程。
3. 左栏：基本信息、运维人员、定位设置、其他设置。
4. 右栏：支付方式、显示器画面、点位照片、操作按钮。

## 4. 结构方案
- 在 header 下新增 `form-grid` 容器。
- `form-grid` 包含 `left-column` 与 `right-column`。
- 原有 `section-title + rows` 以 section 卡片形式迁移至对应列。
- 保持 `deviceSearchInput`、`locationSelect`、`locationAddressInput` 等关键节点不变。

## 5. 样式方案
### 5.1 桌面端
- `form-grid` 双列（主列 + 侧列）布局。
- 每个 section 使用统一卡片样式：边框、圆角、阴影、分组标题。
- 统一行高、label 最小宽度、value 对齐与输入宽度。
- 上传区在桌面保持并排展示。

### 5.2 移动端
- `form-grid` 回落单列。
- row 改为纵向堆叠（label 在上、value 在下）减少挤压。
- 输入框统一左对齐并宽度 100%。
- 上传区支持横向滚动，避免内容溢出。
- 按钮区保持双按钮并提升点击舒适度。

## 6. 风险与规避
- 风险：DOM 重排导致脚本引用失效。  
  规避：不修改关键元素 ID，仅改变容器层级。
- 风险：移动端断点下表单错位。  
  规避：明确 `@media` 下 row/value/input 的覆盖样式。
- 风险：样式覆盖影响其他页面。  
  规避：仅在 `device-entry.html` 内新增样式，不复用全局样式文件。

## 7. 验收标准
1. 桌面端（1440/1920）呈现两栏卡片化布局。
2. 移动端（390）保持单栏可读、可点、无横向整体溢出。
3. 设备搜索下拉、点位联动、提交入场流程均可用。
4. 页面无关键 DOM 丢失（关键 ID 保持可查询）。
