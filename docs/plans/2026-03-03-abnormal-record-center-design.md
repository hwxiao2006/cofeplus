# 全部异常记录中心设计

## 背景
当前 `/Users/tigerhuang/cofeplus/faults.html` 的“异常记录”入口只能从单台设备的“状态记录”进入，无法在故障列表中统一查看全部异常，导致值班排查时需要逐台点开，效率低且容易漏看未恢复异常。

## 目标
- 在故障列表提供“查看全部异常记录”的统一入口。
- 兼容现有单设备“状态记录 > 异常记录”路径，保证历史操作习惯不被破坏。
- 支持按时间、异常类型、状态、设备关键词快速筛选，并突出未恢复异常。

## 非目标
- 本阶段不改造真实后端存储，仅先完成前端原型数据结构与交互。
- 本阶段不引入复杂工单流转（派单、SLA 统计、通知升级）。

## 方案对比与结论
### 方案 A：故障页顶部按钮 + 全屏弹层
- 优点：改动最小，交付快。
- 缺点：筛选、分页、导出等能力叠加后，弹层复杂度高，不利于持续扩展。

### 方案 B（采纳）：故障页 Tab 增加“异常记录”
- 优点：与现有“正常/缺料/故障”信息架构一致，心智统一；可复用现有搜索和内容区；后续扩展字段成本低。
- 缺点：需要在同页区分“设备卡片视图”和“异常记录视图”两套渲染逻辑。

### 方案 C：独立页面 `abnormal-records.html`
- 优点：扩展性最好，长期可演化为独立监控中心。
- 缺点：新增导航层级与维护成本，短期收益不如方案 B。

## 信息架构
- 一级入口：`faults.html` 的状态 Tab 增加 `异常记录`。
- 二级入口：保留设备卡片中的 `状态记录 -> 异常记录`，用于单设备上下文查看。
- 视图关系：
  - `currentTab !== "abnormal"`：维持设备卡片列表。
  - `currentTab === "abnormal"`：切换到“全量异常记录列表”视图。

## 页面与交互设计
### 1) Tab 与概要区
- Tab 调整为：`正常`、`缺料`、`故障`、`异常记录`。
- 概要区文案：
  - 设备视图：`总计 X 台设备`
  - 异常视图：`总计 X 条异常（未恢复 Y 条）`

### 2) 异常记录列表
- 桌面端采用表格风格，移动端采用卡片风格。
- 核心字段：时间、设备编号、点位、异常类型、故障码、异常描述、状态、持续时长。
- 默认排序：按 `startedAt` 倒序。
- 支持点击行查看详情（复用当前 overlay 机制）。

### 3) 筛选与检索
- 时间范围（默认近 7 天）。
- 异常类型（故障/失联/缺料/人工操作）。
- 状态（未恢复/已恢复）。
- 关键词（设备编号/点位/异常描述）。
- 快捷开关：`仅看未恢复`。

### 4) 单设备联动
- 从设备卡片进入“状态记录 > 异常记录”时，自动携带 `deviceId` 过滤条件，只展示该设备记录。
- 从全量异常视图点击某条记录，可跳转回对应设备卡片并高亮（后续可选能力）。

## 数据模型设计（前端原型）
新增 `abnormalRecords` 数据集，单条结构：

```js
{
  recordId: "ABN-20260303-0001",
  deviceId: "RCK019",
  siteName: "COFE+徐汇区行政服务中心*",
  type: "fault",            // fault | lost | shortage | manual
  code: "E-ICE-001",
  message: "制冰机电流异常",
  status: "open",           // open | recovered
  startedAt: "2026-03-03 09:40:28",
  recoveredAt: "",
  recoveredBy: "",
  source: "system"          // system | operator
}
```

派生字段：
- `duration`：`status=open` 时用当前时间减 `startedAt`，`recovered` 时用 `recoveredAt - startedAt`。
- `typeLabel`、`statusLabel`：用于 UI 显示。

## 数据流与状态变更
1. 设备状态变化/告警事件写入 `abnormalRecords`。
2. 同一设备同类型短时间重复告警可合并计数（MVP 可先不合并）。
3. 告警恢复时更新对应记录 `status=recovered` 与 `recoveredAt`。
4. “异常记录”Tab 读取全量记录并执行筛选、排序、分页渲染。
5. 单设备异常页通过 `deviceId` 进行二次过滤。

## 异常与边界处理
- 无数据：显示空状态及“调整筛选条件”提示。
- 时间或故障码缺失：展示 `-`，避免布局抖动。
- 接口失败（后续接 API）：显示错误条与“重试”按钮。
- 超长描述：列表中 1 行省略，详情页展示全文。

## 权限建议
- 运维角色：可查看全量异常、可标记恢复状态。
- 只读角色：可查看与筛选，不可修改状态。
- 本原型阶段先不做鉴权逻辑，仅预留按钮显隐位。
