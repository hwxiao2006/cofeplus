# Product Order And Business Tags Design

## Context

菜单管理当前已经支持分类排序、商品多分类归属、商品详情多语言编辑和点单屏预览，但缺少两项运营侧高频能力：

- 不能在单个分类内调整商品顺序。
- 商品缺少可复用、可多语言、可多选的业务标签体系。

同时，现有 `推荐商品` 是单一布尔开关，既不支持多标签运营，也和后续的业务标签体系重复。顶部统计区也存在信息过多和统计口径不清的问题。

## Approved Scope

本次设计锁定以下范围：

1. 支持在单个分类内调整商品顺序。
2. 商品支持绑定多个业务标签，标签来自全局标签库，支持多语言。
3. `推荐商品` 功能整体下线，由业务标签接替原有前台展示位。
4. 菜单管理顶部统计卡只保留 `商品总数` 和 `在售商品数`。
5. 两项统计都按 `商品 ID 去重` 计算，避免多分类商品重复计数。

本次不包含：

- 按业务标签做顶部汇总卡片
- 按业务标签做商品筛选
- 业务标签参与价格、配方、默认项或上下架逻辑

## Decision Summary

### 1. 分类内商品顺序

- 顺序以 `当前分类的商品数组顺序` 为准，不增加新的全局商品顺序概念。
- 同一个商品出现在多个分类时，每个分类独立排序，互不影响。
- 调整顺序只允许在 `当前分类` 视角下进行；`全部分类` 视角不提供排序入口。
- 新加入某分类的商品默认插入该分类末尾。

### 2. 全局业务标签库

- 业务标签采用 `全局标签库` 模式，不采用“每个商品单独创建标签”的模式。
- 每个标签至少包含：
  - `id`
  - `names`（按语言存储）
  - `status`（`active` / `disabled`）
- 商品本身只保存已绑定标签的有序 ID 列表，例如：

```js
businessTagIds: ['tag_recommend', 'tag_new']
```

- 数组顺序即展示优先级，不额外引入第二套排序字段。

### 3. 推荐商品功能下线

- 删除商品详情页和商品弹窗中的 `推荐商品` 开关。
- 删除菜单管理商品卡、点单屏预览商品卡上的 `推荐` 徽标。
- 删除顶部统计区中的 `推荐商品` 统计卡。
- 旧数据中的 `featured` 字段保留兼容，但不再展示、不再编辑、不再参与统计。

### 4. 业务标签接替前台展示

- 原来依赖 `推荐商品` 的前台展示位，统一改由业务标签承担。
- 如果商户需要表达“推荐 / 热卖 / 新品”等运营信息，应通过业务标签配置完成。

## Interaction Design

### A. 调整商品顺序

#### 桌面端

- 入口文案：`调整商品顺序`
- 入口位置：商品管理当前分类的商品区标题栏
- 点击后：右侧商品卡片区切换为紧凑排序列表，不新开页面
- 列表行信息：
  - 拖拽手柄
  - 商品缩略图
  - 商品名称
  - 当前序号
- 顶部固定：
  - 标题：`调整“{分类名}”商品顺序`
  - 说明：`拖动商品，可调整它在当前分类中的显示顺序`
  - 操作：`取消` / `保存`

#### 移动端

- 同样使用 `调整商品顺序` 入口
- 点击后进入全屏排序页，而不是在原卡片区内硬切换
- 使用大尺寸拖拽手柄，底部固定 `取消` / `保存`
- 保存后回到原分类商品列表

### B. 业务标签编辑

- 删除原 `功能开关 > 推荐商品`
- 在 `商品详情 > 基本信息` 中增加 `业务标签` 模块
- 模块默认展示当前已选标签，并提供 `编辑` 入口
- 编辑界面需要支持：
  - 多选已有标签
  - 新建标签
  - 编辑标签
  - 拖动调整已选标签顺序
- 标签新增/编辑时：
  - 当前设备默认语言必填
  - 其他已启用语言可选填

### C. 停用标签

- 标签停用后：
  - 不再允许新商品选择
  - 所有商品不再显示该标签
  - 点单屏商品卡和点单屏商品详情均不显示该标签
- 底层商品绑定关系可保留，便于标签重新启用后恢复展示

## Display Rules

### 1. 菜单管理商品卡

- 显示前 2 个启用中的业务标签
- 超过 2 个时显示 `+N`
- 标签顺序使用 `businessTagIds` 的数组顺序

### 2. 商品详情页

- 显示全部启用中的业务标签
- 支持在编辑状态下直观看到标签顺序和展示结果

### 3. 点单屏商品卡

- 不再显示 `推荐` 徽标
- 改为在商品名下方单独展示一行业务标签
- 最多显示前 2 个启用中的标签
- 不显示 `+N`
- 目的是保证点单屏列表稳定，不被过多标签撑高

### 4. 点单屏商品详情

- 在商品详情顶部摘要区展示全部启用中的业务标签
- 放置顺序：商品名 / 价格之后，描述之前
- 以小型标签 chip 展示，不使用强提醒徽标样式

## Data Model

### 商品分类内顺序

保持现有 `productsData[categoryKey].items[]` 结构，直接以数组顺序表达该分类内的商品顺序。

```js
productsData['奶咖系列'].items = [
  { id: 101, ... },
  { id: 205, ... },
  { id: 302, ... }
]
```

### 全局标签库

建议新增共享标签库状态，例如：

```js
businessTagLibrary = [
  {
    id: 'tag_recommend',
    names: { zh: '推荐', en: 'Recommended' },
    status: 'active'
  },
  {
    id: 'tag_new',
    names: { zh: '新品', en: 'New' },
    status: 'disabled'
  }
]
```

### 商品标签字段

```js
{
  id: 101,
  names: { zh: '生椰拿铁' },
  businessTagIds: ['tag_recommend', 'tag_new']
}
```

说明：

- `businessTagIds` 同时承担“绑定关系”和“展示顺序”两种职责。
- 商品的业务标签属于商品基础信息，修改后需要同步到所有同 ID 商品实例。

## Stats Rules

顶部统计卡仅保留：

- `商品总数`
- `在售商品数`

统计口径：

- 以 `商品 ID` 去重后统计
- `商品总数` = 去重后的全部商品数
- `在售商品数` = 去重后的在售商品数

## Compatibility

- 旧商品没有 `businessTagIds` 时按空数组处理。
- 旧系统中的 `featured` 字段继续保留在数据里，但所有新逻辑忽略它。
- 旧分类商品列表没有额外排序字段，不需要迁移；现有数组顺序即默认排序结果。

## Validation Scope

至少覆盖以下场景：

1. 在分类 A 中调整商品顺序后，分类 B 中同 ID 商品顺序不变。
2. 商品加入新分类时默认进入末尾。
3. 商品可以绑定多个业务标签，并保持选择顺序。
4. 标签支持多语言显示。
5. 标签停用后，菜单管理、商品详情、点单屏商品卡、点单屏商品详情全部不再显示。
6. 点单屏商品卡只显示前 2 个标签且不显示 `+N`。
7. 顶部统计卡只保留商品总数和在售商品数，且按商品 ID 去重。
8. `推荐商品` 的开关、徽标、统计入口在所有主页面中都被移除。

## Implementation Notes

- 优先复用现有“同 ID 商品同步更新”的保存逻辑，不重新设计全局商品主表。
- 业务标签的视觉样式使用统一 chip，不再沿用强强调的橙色 `推荐` 徽标。
- `menu.html`、`overview.html` 以及相关 figma/mock 页面如果仍被团队使用，需要同步移除 `推荐商品` 相关展示，避免页面语义不一致。
