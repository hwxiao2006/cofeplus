设备详情页布局方案

方向已锁定为 运营优先。下面 3 个方案都遵守同一原则：首屏先回答“这台设备是谁的、现在是否正常、最常用动作是什么”，深层信息后置。

推荐目标

  • 普通运营第一次进来也能 5 秒内看懂
  • 远程操作仍保留为设备级通用动作
  • 故障、入场、图片、记录都存在，但不抢首屏

必须避免

  • 一打开就出现 5 个大块卡片并列竞争注意力
  • 目录导航和内容本身同时抢用户眼睛
  • 动作入口散落在多个区域
A. 概览首屏 + 动作侧栏
设备编号 / 商户 / 点位 / 当前状态
首屏概览卡：设备状态 / 停卖 / 最近心跳 / 入场状态
当前异常摘要：只有异常时才高亮
可折叠区：入场核心 / 入场详情 / 图片 / 完整记录
统一设备操作
远程操作 / 编辑状态 / 状态记录 / 物料页面

A. 概览首屏 + 动作侧栏

推荐。 最像一个“人人看得懂”的设备名片页。先看概要，再决定是否展开故障或入场细节。

B. 三段式主内容
第一段：设备名片 + 关键状态 + 常用动作
第二段：异常与运维信息
第三段：入场、图片、技术细节

B. 三段式主内容

没有明显侧栏，阅读路径更顺，但动作区会往下沉一点，桌面端“操作感”略弱。

C. 概览首屏 + 二级 Tab
设备概览
首屏固定：基础信息 + 状态 + 动作
点 Tab 才切换对应内容

C. 概览首屏 + 二级 Tab

最干净，但会把信息藏得比较深，适合重度使用者，不太适合第一次进入的普通运营。

我的推荐

我推荐 A。因为它既保留设备级统一动作入口，又把“看信息”和“做动作”拆清楚了，而且不需要做双模式，后续从故障页跳进来时也最自然。
